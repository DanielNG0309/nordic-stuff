# Multilateration System for Position Estimation

## Overview

This Python application (`multilateration.py`) provides a real-time position tracking system designed to process distance measurements, such as those generated by Bluetooth Channel Sounding using Nordic Semiconductor nRF54L15 chips. The script estimates the position of a target object (Reflector) in a 2D space using distances measured from several fixed reference points (Anchors).

The system employs advanced algorithms—RANSAC and a Kalman Filter—to ensure robust and smooth tracking, even when individual distance measurements are noisy or contain errors.

The application can operate in two modes:
1.  **Simulation Mode:** Generates synthetic movement and distance data to test the algorithms and demonstrate the visualization without hardware.
2.  **Real-time Mode:** Reads live distance data from connected hardware devices via COM ports.

## Key Features

- **Dual Mode Operation:** Supports both simulation for demonstration and real-time hardware integration.
- **Robust Estimation (RANSAC):** Effectively identifies and ignores incorrect distance measurements (outliers) to calculate a reliable position fix.
- **Smooth Tracking (Kalman Filter):** Provides smooth visualization of the object's path and reduces jitter.
- **Interactive Visualization:** A live plot shows anchor positions, the estimated object position, and visual representations of the distance measurements.
- **Configuration Management:** A user-friendly interface for setup, which automatically saves and loads the configuration (anchor positions, COM ports, parameters).
- **Live Tuning:** Key parameters can be adjusted while the system is running to optimize performance during a demonstration.
- **Data Logging:** Capability to log raw distance data and quality metrics (RSSI, Samples) to a CSV file for analysis.

## Parameters

### System Parameters
The proper system parameters to use during the real-time channel sounding setup can be selected by choosing "Real-time" under **Mode Selection**, disabling "Accelerometer (Sim Only)" in the **Processing Options** and then selecting "custom" under **System Parameters**. The **System Parameters** modules contains several adjustable toggles and values. Upon launching the configuration screen, our preferred values should already be set, but if for whatever reason the values are wrong, we will also include them here to look up:

- **Basic:**
   - Update Interval: `50`
   - Kalman Filter Q: `2.0`
   - RANSAC Threshold: `0.3`
- **Filtering:**
   - Median Filtering:
      - *Enabled*
      - Filter Window Size: `15`
      - Data Staleness: `1.0`
   - Measurement Gating: 
      - *Disabled*
      - Gating Threshold (if enabled): `9.21`
- **WLS:**
   - *Disabled*
- **Dynamic R:**
   - *Disabled*
- **Asymmetric**
   - *Disabled*
- **Health:**
   - *Disabled*
- **Display:**
   - *All options enabled* (can be tweaked based on personal preference)

### Possible tweaks

The most important values that could be worthwhile to tweak are the following:

#### RANSAC Threshold (m)
By far the most influental tweakable parameter. It determines how aggressively the system rejects bad distance measurements (outliers). This can be adjusted anywhere between `0.15` and `0.50` (can be tuned beyond these values as well, but this is the boundary within which we have typically stayed when testing).
- **Lower values:** Aggressively reject measurements that don't agree with others (stricter filtering). If set too low in a noisy environment, the system might struggle to find a position.
- **Higher values:** Are more tolerant of noise.

#### Kalman Filter Q
Controls the balance between responsiveness and smoothness.
- **Higher values:** The system reacts faster to changes in direction, but the visualization may appear jittery.
- **Lower values:** Provides a smoother track but may lag slightly during fast movement or sudden direction changes.

#### Other values
Beyond the RANSAC threshold, the median filter can be adjusted to get smoother movements on the display or lower for more responsive ones. Measurement gating should also in theory help with more sensical position estimates based on what is statistically probable. We haven't found much of a difference in having this enabled or disabled, but your milage may vary.

### Anchor Selection

Anchors need to be defined by their com port and their coordinates. Simply place them in relation to each other using coordinates in such a way that they geometrically match their placement in real life. We generally have used A0 as coordinate (0,0) and then placed the rest of the anchors in a rectangle, sometimes with two additional anchors in the middle. Linking them to the correct COM ports can be done by using the VSCode extension nRF Connect, navigating down to the "connected devices" tab, and writing down the com ports corresponding to the anchor's device name (VCOM1 will be the correct one of the two possible options).
